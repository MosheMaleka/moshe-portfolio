Business Problem

The organization lacked a centralized view of sales performance. Sales data existed across multiple Excel spreadsheets which required manual consolidation and analysis.

  • Reporting took 5–7 days each month
  • No centralized KPI visibility
  • Limited insight into regional performance
  • Manual processes introduced reporting errors

Solution Approach

1. Data Extraction

Raw sales data was extracted from Excel files and imported into a Python-based ETL pipeline developed in Google Colab.

2. Data Transformation

Using Python (Pandas), the dataset was cleaned and standardized:

  • Removed null and duplicate records
  • Standardized region naming conventions
  • Converted order dates to datetime format
  • Calculated Net Revenue (Revenue – Discount)
  • Created Month and Year attributes
  • Computed Margin %
3. Data Storage

The cleaned dataset was exported and loaded into PostgreSQL hosted on Supabase, enabling structured SQL analysis.

4. Analytics & Visualization

Power BI Desktop was deployed on a Windows Virtual Machine hosted on AWS EC2. This allowed dashboard development in a cloud environment without requiring a local Windows system.

Cloud Architecture

The analytics solution uses a cloud-based data pipeline that processes raw data, transforms it using Python, stores it in PostgreSQL, and visualizes insights through Power BI running on an AWS virtual machine.

Key Insights

  • One region exceeded targets but had declining margins
  • Underperforming reps lacked product diversification
  • 60% of revenue originated from a seasonal product category
  • Revenue leakage was caused by inconsistent discounting

Business Impact

  • Reduced reporting time from 5–7 days to under 1 day
  • Enabled proactive sales performance monitoring
  • Improved executive visibility of revenue trends
  • Supported strategic decision-making using real-time insights
